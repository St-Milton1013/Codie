# Phase 12P - Optional Outbound Delivery Contract

## Purpose

Define the safety contract for any future outbound delivery feature such as
Discord, email, cloud upload, file sync, or public link creation.

This is a planning-only packet. It does not add outbound delivery code.

## Delivery Classes

### Manual Delivery

Already allowed:

```text
copy files manually
move generated PDF manually
open bundle on LAN preview manually
```

No credentials or network automation are involved.

### Local LAN Preview

Already implemented:

```text
serve-share-bundle
```

This serves one selected bundle read-only. It does not send files outward.

### Outbound Delivery

Deferred:

```text
Discord webhook
email
cloud upload
file sync provider
public link generation
URL shortener
```

Any outbound delivery requires a separate implementation contract after this
planning gate.

## Required Defaults

Future outbound delivery must default to:

```text
enabled = false
dry_run = true
send_without_preview = false
store_credentials = false
create_public_link = false
```

No command may send a report unless the user explicitly supplies a destination
and disables dry-run behavior.

## Required Workflow

Future outbound delivery must follow this order:

```text
1. Build local share bundle.
2. Select outbound provider explicitly.
3. Load credentials from environment or ignored local config.
4. Produce payload summary.
5. Run dry-run.
6. Display destination, files, byte sizes, and privacy warning.
7. Send only when user passes an explicit send flag.
8. Return delivery result without logging secrets.
```

## Credential Rules

Credentials:

```text
must come from environment variables or ignored local config
must never be committed
must never be printed
must never be written to reports
must never be embedded in QR codes
must be redacted in errors
```

Missing credentials must fail cleanly before network work begins.

## Provider-Specific Requirements

### Discord Webhook

Future command may be:

```text
send-share-bundle-discord
```

Required:

```text
webhook URL from environment variable or ignored config
dry-run default
explicit --send flag for real send
payload summary before send
file size guardrail
no credentials in logs
tests mock network calls
```

Forbidden:

```text
committed webhook URL
automatic posting
posting private reports without explicit destination
posting source database files
```

### Email

Required:

```text
explicit recipient
dry-run default
credential source documented
attachment size guardrail
tests mock network calls
```

### Cloud Upload / Public Link

Requires extra review before implementation.

Additional requirements:

```text
explicit privacy warning
explicit retention policy
explicit visibility setting
no default public link
no URL shortener by default
```

## Payload Rules

Allowed payloads:

```text
share bundle zip, if generated by a future packet
single generated PDF
manifest.json
index.html
print.html
selected report assets
```

Forbidden payloads:

```text
SQLite database files
raw provider payload archives
credentials
ignored config files
unreviewed local directories
```

## Test Requirements For Future Implementation

```text
dry-run does not call network
send requires explicit flag
missing credentials fail before network call
credentials are redacted in errors
destination is required
payload summary includes files and byte sizes
oversized payload fails cleanly
network calls are mocked
SQLite files are rejected
ignored config files are rejected
```

## Required Static Scans

Future implementation must scan for:

```text
webhook URL literals
committed tokens
sqlite database payload inclusion
provider/db/analytics/recommendation imports in delivery modules
raw source/provider table reads
```

## Recommended Future Packets

```text
Phase 12Q - Share Bundle Zip Export Contract
Phase 12R - Share Bundle Zip Export Implementation
Phase 12S - Discord Delivery Contract
Phase 12T - Discord Delivery Implementation
```

Discord should not be implemented until bundle zip/export packaging exists.

## Do Not Do In Phase 12P

- Do not add outbound delivery code.
- Do not add dependencies.
- Do not add webhooks.
- Do not add email/cloud integrations.
- Do not create public links.
- Do not generate zip files.
- Do not change schema.
- Do not change recommendation logic.
