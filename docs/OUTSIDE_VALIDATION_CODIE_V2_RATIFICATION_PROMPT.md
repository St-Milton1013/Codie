# Outside Validation Prompt — Codie V2 Constitution Ratification

Validate the V2 adoption packet as a governance-only change.

## Required review files

```text
docs/CODIE_V2_CONSTITUTION.md
docs/CODIE_V2_CHANGELOG.md
docs/CODIE_V2_COMPATIBILITY_STATEMENT.md
docs/CODIE_V2_RATIFICATION_CONTRACT.md
docs/CHECKPOINT_CODIE_V2_RATIFICATION_REPORT.md
docs/CODIE_V1_CONSTITUTION.md
docs/ACTIVE_ROADMAP_INDEX.md
docs/VALIDATION_STATUS_INDEX.md
docs/NEXT_PHASE_CONTRACT.md
docs/CODEX_CONTINUITY_HANDOFF.md
```

## Required determinations

Confirm that:

1. V2 is internally consistent and records the explicit ratified decisions.
2. V1 is retained unchanged and clearly marked historical after adoption.
3. V2 preserves canonical-data, provenance, privacy, zero-cost, repository, Evidence Fusion, Decision Intelligence, and validation boundaries.
4. Jin cannot mutate measured evidence or persisted recommendations.
5. Relationship Intelligence is metric-only and exposes formulas, populations, counts, provenance, and caveats.
6. Tutor-pile best-line results cannot be mislabeled deterministic.
7. Local model execution remains viable and cloud data egress is profile- and data-class-controlled.
8. Unknown and abstention states cannot silently collapse into zero or false.
9. All V1 capabilities carry forward except the two explicit carry-forward exceptions.
10. The packet does not advance Phase 37 or authorize runtime work.

## Reject for

- deletion or modification of V1;
- direct runtime, schema, provider, workflow, validator, or dependency changes;
- unresolved conflict between the constitution, change log, and compatibility statement;
- wording that makes the branch authoritative before merge;
- wording that treats roadmap inclusion as implementation authorization;
- loss of source provenance, uncertainty, privacy, or recommendation boundaries.

## Outcomes

```text
PASS
PASS WITH REVIEW NOTES
PASS WITH REQUIRED FIXES
FAIL
```

Only PASS or PASS WITH REVIEW NOTES permits adoption and merge.
